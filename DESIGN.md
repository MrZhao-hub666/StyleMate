# StyleMate — 设计文档

> ⚠️ 本文档记录项目**原始设计思路**，与当前实际实现可能有出入。当前架构请参考 [ARCHITECTURE.md](ARCHITECTURE.md)。

**一句定位**：以 YOLO 边端视觉解析为感知基石，以云端大模型为推理引擎，结合 RAG 知识库与 AI 生图能力的智能穿搭顾问系统。

---

## 一、系统概述

### 1.1 项目背景

随着人工智能技术的快速发展，边缘计算与云计算的协同工作已成为解决大规模数据处理、低延迟响应和复杂智能决策的关键架构。StyleMate 将这一思想应用于日常穿搭场景，构建了一套完整的 **"端-边-云"三层协同**智能穿搭系统。

### 1.2 核心设计理念

**"边端做感知，云端做决策，知识库做约束，生图做呈现"**

| 层级 | 角色 | 比喻 |
|------|------|------|
| 边端 YOLO | 精准的眼睛 | 像一位细心的观察者，把每件衣服的结构化属性全部"看"清楚 |
| 云端 Agent | 聪明的大脑 | 基于精确数据做逻辑推理，而非看图瞎猜 |
| 知识库 RAG | 专业的顾问 | 提供色彩搭配理论、场合着装规范等专业知识 |
| AI 生图 API | 可视化呈现 | 把抽象的搭配方案变成用户能直观看到的效果图 |

### 1.3 设计哲学

- **不依赖大模型看图理解衣物** — 所有衣物属性由 YOLO 边缘解析引擎结构化提取，Agent 在精确数据上推理，避免大模型"看图猜衣服"的不稳定性
- **知识驱动而非数据驱动** — 系统内建时尚专业知识库，LLM 做推理时受知识约束，输出有据可依
- **模块解耦、职责分明** — 每个模块只做自己擅长的事，通过标准化 API 通信，可独立升级替换

## 二、核心功能

### 功能A：智能衣橱 + 搭配推荐（YOLO 重兵把守）

#### A1 — 衣物入库（建数字衣橱）

用户将多件衣物拍照或摄像头扫描 → 边端 YOLO 对每件衣物执行**七维属性解析**：

| 维度 | 解析内容 | 技术手段 |
|------|----------|----------|
| ① 品类细分 | 20+ 细分类：圆领T恤/衬衫/卫衣/夹克/西装/牛仔裤/短裤/裙子... | YOLOv8-seg 检测分割 + YOLOv8-cls 分类 |
| ② 颜色提取 | 主色 HEX + 辅色 HEX + 各色占比 | OpenCV HSV 空间 K-means 聚类 |
| ③ 图案识别 | 纯色 / 条纹 / 格子 / 碎花 / 印花 / 迷彩 / 渐变 | Gabor 滤波器频谱分析 |
| ④ 款式特征 | 领型(圆领/V领/翻领/立领)、袖长(无袖/短袖/长袖)、版型(修身/标准/宽松) | YOLOv8-cls 细粒度分类 |
| ⑤ 长度比例 | 上装(露脐/及腰/盖臀/中长)、下装(超短/短款/及膝/全长) | 分割 mask 比例计算 |
| ⑥ 材质推测 | 棉/麻/丝/毛/牛仔/皮质/针织/化纤 | GLCM 纹理特征 + 光泽度分析 |
| ⑦ 图像分割 | 透明背景的纯衣物抠图 | YOLOv8-seg 实例分割 |

每件衣物入库后存储为结构化 JSON 档案，形成用户的**数字衣橱**。

#### A2 — 智能搭配推荐

```
用户输入：今天要去【面试】+ 偏好【沉稳大气】
系统自动获取：当前天气【22°C 多云】

Agent 推理六步：
  ① 天气筛选 → 温度匹配的衣物范围
  ② 场合匹配 → 知识库检索面试着装规范
  ③ 色彩搭配 → 从衣橱中找主色协调的上下装组合
  ④ 款式搭配 → 比例法则(上短下长/上宽下窄...)
  ⑤ 生成方案 → 输出 2-3 套穿搭推荐 + 每套理由
  ⑥ 生图预览 → 每套方案调用通义万相生成穿搭效果参考图
```

#### A3 — 上身拍照直接识别

用户穿好某套搭配站在镜头前 → YOLO 解析当前全身穿搭的结构化数据 → 可直接衔接功能 B 做评价，也可基于当前某件单品推荐更多搭配。

---

### 功能B：穿搭评价与建议

```
用户穿好衣服 → 站镜头前拍照 → YOLO 解析全身穿搭
   → 用户说明场合（如"今天要去相亲"）
   → Agent 分析：
       ├─ 配色协调性（知识库：色彩搭配理论）
       ├─ 场合匹配度（知识库：场合着装规范）
       ├─ 款式搭配度（知识库：穿搭黄金法则）
       └─ 整体完成度
   → 输出：
       ├─ 综合评分（1-10分）
       ├─ 亮点分析
       ├─ 不足之处
       ├─ 逐条改进建议
       └─ 可选：生成改进后的效果预览图
```

---

### 功能C：肖像/穿搭创意生成

双模式切换，满足不同需求：

| 模式 | 采集方式 | 适用场景 | 生图方式 |
|------|----------|----------|----------|
| **肖像模式** | 半身/头像照 | 职业证件照、社交头像、古风写真、赛博朋克 | 图生图：保留面部特征，风格迁移 |
| **全景模式** | 全身照 | 穿搭街拍、场景化写真、T台秀、旅行度假风 | 图生图：全身人物+背景+风格合成 |

```
用户操作流程：
  ① 选择模式（肖像 / 全景）
  ② 拍照或上传已有照片
  ③ 输入场景描述或选用预设模板（如"日系校园风樱花树下"）
  ④ Agent 优化 prompt → 通义万相生图
  ⑤ 返回创意写真/穿搭大片
```

## 四、技术栈详表

### 4.1 前端

| 技术 | 用途 | 选型理由 |
|------|------|----------|
| Vue 3 (Composition API) | UI 框架 | 响应式、生态成熟 |
| Vite | 构建工具 | 极速 HMR，开发体验好 |
| Element Plus | 组件库 | Vue 3 生态最成熟，组件丰富 |
| Pinia | 状态管理 | Vue 3 官方推荐，TypeScript 友好 |
| Vue Router | 路由 | 多页面导航 |
| WebRTC (getUserMedia) | 摄像头采集 | 浏览器原生，零依赖 |
| Canvas API | 衣物框选/裁剪 | 原生高性能 |
| axios | HTTP 请求 | 拦截器、超时、取消均完善 |
| WebSocket | 实时通信 | 生图结果实时推送 |

### 4.2 云端后端

| 技术 | 用途 | 选型理由 |
|------|------|----------|
| Python 3.11 | 运行环境 | AI 生态首选 |
| FastAPI | Web 框架 | 异步原生、自动文档、WebSocket 内置 |
| SQLAlchemy 2.0 | ORM | 异步支持、成熟稳定 |
| SQLite | 数据库 | 单用户轻量场景，零运维 |
| Nginx | 反向代理 + 静态资源 | 高性能、配置简单 |
| Docker + docker-compose | 容器化 | 环境一致性，一键部署 |
| Pydantic | 数据校验 | FastAPI 原生集成 |

### 4.3 Agent 智能体

| 技术 | 用途 | 选型理由 |
|------|------|----------|
| LangGraph | 功能A 图编排 | 并行分支 + 条件路由 + 多路汇聚，链式框架无法胜任 |
| LangChain | 功能B/C Agent | 工具调用封装、RAG 检索器、Memory 管理 |
| DeepSeek API | 大语言模型 | 性价比极高、中文能力强、兼容 OpenAI 格式 |

### 4.4 知识库 (RAG)

| 技术 | 用途 | 选型理由 |
|------|------|----------|
| ChromaDB | 向量数据库 | 轻量嵌入式、Python 原生、零运维 |
| text2vec-large-chinese | Embedding 模型 | 开源中文语义向量，免费本地运行 |
| LangChain Retriever | 检索接口 | 统一检索抽象 |

### 4.5 外部 API

| API | 用途 | 价格 |
|------|------|------|
| 通义万相 | 文生图 / 图生图（穿搭预览+创意写真） | 有免费额度 |

| Tavily Search | 联网搜索时尚资讯 | 有免费额度 |

### 4.6 边端 (YOLO 解析引擎)

| 技术 | 用途 | 选型理由 |

| scikit-learn (K-means) | 主色聚类提取 | 成熟稳定 |
| scikit-image (Gabor/GLCM) | 纹理图案分析 | 科学计算社区标准 |
| requests / aiohttp | 数据上传 HTTP 客户端 | 简单可靠 |

## 六、用户使用流程

### 6.1 新用户首次使用

```
1. 打开系统 → 进入首页
2. 首次使用引导：拍摄/上传衣柜中的衣物
3. 边端 YOLO 自动解析每件衣服的属性
4. 衣物入库完成 → 形成个人数字衣橱
5. 现在可以使用三个功能模块
```

### 6.2 日常使用

```
场景一（需要搭配建议）：
  进入功能A → 输入场合和偏好 → 系统自动获取天气
  → 等待 Agent 分析 → 获得 2-3 套推荐 + 预览效果图
  → 选择喜欢的搭配 → 照着穿 ✓

场景二（穿好后寻求评价）：
  进入功能B → 穿好衣服站镜头前 → 输入场合
  → 等待评价 → 获得评分 + 亮点 + 不足 + 改进建议
  → 根据建议调整 ✓

场景三（想玩创意）：
  进入功能C → 选择肖像/全景模式 → 拍照或上传
  → 输入创意描述 → 等待生图 → 获得 AI 创意写真 ✓
```

## 七、非功能性特性

| 特性 | 实现方式 |
|------|----------|
| **实时性** | YOLOv8n 引擎在 CPU 上可达 20+ FPS 检测帧率 |
| **响应性** | 边端检测 < 100ms，云端 Agent 推理 < 5s，生图 < 15s |
| **可扩展性** | Agent 工具注册机制，新增工具只需实现接口 |
| **稳定性** | 前端断网提示，后端请求重试机制 |
| **易用性** | B/S 架构，浏览器打开即用，无需安装客户端 |

## 九、设计决策记录

| 决策 | 选择 | 放弃 | 理由 |
|------|------|------|------|
| Agent 框架 | LangGraph（A）+ LangChain（B/C） | 纯 Chain / 全部手写 | 功能A需要并行+条件+汇聚，链式不够；B/C简单，Agent够用 |
| LLM | DeepSeek API | 本地部署开源模型 | 2C2G 无法本地跑 LLM，API 成本极低 |
| 生图 | 通义万相 API | Stable Diffusion 本地部署 | 2C2G 跑不动 SD，API 效果好 |
| 边云通信 | HTTP REST | MQTT / WebSocket | 边端只做上传，无实时双向需求，HTTP 最简单可靠 |
| 前端 | Vue 3 + Element Plus | React / Streamlit | Element Plus 组件丰富开箱即用，适合快速开发 |
| 数据库 | SQLite | PostgreSQL / MySQL | 单用户场景，零运维更合适 |
| 衣物解析 | YOLO分区+颜色HSV分析+纹理CV -> 云端LLM语义理解（边云混合方案B） | 纯大模型看图 | 纯大模型看图不稳定且不可控；边端YOLO做精确分区和mask裁剪，颜色用HSV命名（覆盖全色彩），结构数据交给云端LLM做语义理解，既保证可控又有语义深度 |
| 颜色命名 | HSV色彩空间自动计算 | 硬编码颜色字典 | 之前30色字典覆盖面太窄导致"深黑标成藏青"等问题，HSV方案覆盖所有颜色，输出"深蓝色"/"亮红色"等自动生成名 |
| 虚拟试穿 | AI 生图效果预览 | 真正的 VITON 模型 | VITON 太重且效果不稳定，生图预览满足课程需求 |
